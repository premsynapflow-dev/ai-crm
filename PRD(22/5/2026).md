# SynapFlow — Product Requirements Document
**Generated from codebase analysis · May 2026**

---

## 1. Product Overview

**SynapFlow** is a multi-tenant, AI-powered complaint intelligence SaaS platform targeting Indian fintechs, NBFCs, D2C brands, and e-commerce SMBs. It automates complaint ingestion, classification, routing, and resolution, with a gated RBI compliance module as the primary enterprise upsell.

**Stack:** FastAPI (Python) · Next.js + Tailwind · PostgreSQL via Supabase · Gemini AI · Razorpay

## 3. Features Built & Visible on Frontend

### 3.1 Complaints Inbox (`/complaints`)
- List view with filter/sort by status, priority, sentiment, category
- Complaint detail modal: message history, sentiment badge, category, urgency score
- Status transitions (New → Open → Resolved) with audit trail
- Manual reply sending from modal
- Bulk actions (assign, close)

### 3.2 Dashboard (`/dashboard`)
- KPI summary cards: total tickets, open, resolved, CSAT trend
- Volume trend chart, priority breakdown
- Quick-action links to inbox and reply queue

### 3.3 Customer 360 (`/customers`, `/customers/[id]`)
- Customer list with search/filter
- Individual profile: full complaint history, sentiment label, churn risk badge
- Health scoring indicators (VIP, risky, billing-watch, renewal-ready)
- Aggregated interaction timeline per customer

### 3.4 Assignments (`/assignments`)
- Team & agent workload view: open ticket counts per member
- Manual ticket reassignment via dropdown
- Capacity tracking table (agent → ticket load)

### 3.5 Teams (`/settings/teams`)
- Create/manage teams and members
- View team routing configuration

### 3.6 AI Reply Queue (`/reply-queue`)
- Pending AI-generated draft replies
- Approve / Edit / Reject workflow (HITL)
- Confidence score display per draft
- Tabbed by status: pending, approved, rejected

### 3.7 RBI Compliance (`/compliance`) — *Scale/Enterprise only*
- TAT status per complaint (within_tat / approaching_breach / breached)
- RBI category taxonomy display
- MIS report generation trigger
- Upgrade prompt shown for lower plans

### 3.8 Analytics (`/analytics`) — *plan-gated*
- Volume trend (area chart), priority breakdown (bar chart), category distribution (pie chart)
- Churn risk customers list (Max+)
- Root cause analysis summary (Max+)
- Team performance table (Max+)
- Upgrade prompts for locked sections

### 3.9 Billing & Plans (`/pricing`)
- Plan comparison table, current plan indicator
- Upgrade flow (Razorpay checkout)
- Monthly / Annual toggle

### 3.10 Usage & Limits (`/usage`)
- Tickets used vs quota this month
- Overage indicator, billing cycle dates

### 3.11 Settings (`/settings`)
- **Profile tab:** name, email, password change
- **Company tab:** company name, business type (used for RBI eligibility)
- **API tab:** API key display & copy
- **Connections tab:** webhook endpoints (inbound)
- **Webhooks tab:** outbound webhook config
- **Notifications tab:** Slack webhook URL, email alert toggles

### 3.12 Inbox Connections (`/settings/connections`)
- Gmail OAuth connect/disconnect (status badge, connected account)
- WhatsApp (Meta Business API) connect/disconnect
- Email forwarding address (generic SMTP inbound)

### 3.13 Integrations (`/settings/integrations`)
- Same as connections but with additional display context

### 3.14 Live Chat Widget (`/settings/widget`)
- Widget embed code generator (iframe snippet with apiKey param)
- Company name configuration

### 3.15 Admin Panel (`/admin`) — *Internal only*
- Overview: total tenants, active tenants, total tickets, active subscriptions
- Tenant list with plan status
- Manual plan override capability

### 3.16 Public Landing Page (`/`)
- Marketing copy, feature highlights, pricing CTA
- Signup / Login links

### 3.17 Auth (`/login`, `/signup`)
- Email/password signup with business type field (RBI eligibility detection)
- Session-based auth (cookie)

## 8. Data Architecture Highlights

- **Multi-tenant:** All queries scoped by `client_id`; Row-Level Security (RLS) configurable via env
- **Audit log:** Every ticket state change and AI interaction logged to `event_logs`
- **Alembic migrations:** DB version-controlled (active)
- **Background jobs:** In-process SQL-backed thread (no Redis/Celery)
- **JSONB:** Used for flexible per-tenant config, feature flags, audit metadata
